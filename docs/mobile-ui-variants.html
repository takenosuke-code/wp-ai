Mobile UI — 3 variants (all features kept)

Same functionality as today — chat, image upload, live preview, SEO screen, title/schedule/publish flow, history, published list. Only the phone navigation and button visibility change. Pick one (or mix): the main client complaint was that the “next thing to tap” isn’t obvious on the phone.

A — RECOMMENDED

Bottom tabs + one big “next” button

Phone-native tab bar (チャット / プレビュー / 記事). The 8-step bar collapses into one purple progress chip. A sticky button always shows the single next action.
